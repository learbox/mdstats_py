# 常见问题排查

> **最常见的识别问题根源是模板不匹配**——请确保模板图片在当前游戏分辨率下截取。
> 如硬币/先后攻/胜负识别不准，优先重截对应模板再试。

## 识别与检测

### 状态栏一直显示"等待 Master Duel 窗口…"，不开始识别

**原因**：程序无法找到或获取游戏的客户区尺寸。

**检查**：
1. Master Duel 是否已启动？窗口标题需要包含英文 `masterduel`（Yu-Gi-Oh! Master Duel）
2. 窗口是否最小化？恢复窗口再试
3. 分辨率是否 0×0？切换到窗口模式再切回全屏

### 启动时提示"缺少 N 个模板"，状态栏卡住

**原因**：`resource/templates/{分辨率}/` 下缺少必选模板（coin_win/coin_lose/go_first/go_second/victory/defeat）。

**解决**：在对应分辨率目录下放入全部 6 张必选模板。可选模板（rank_up/rank_down）缺失不影响启动。

### 提示"缺少段位检测模板: rank_up.png（已跳过，视为普通局）"

**正常行为**：rank_up/rank_down 是可选模板，缺失时自动跳过，对局照样记录，只是不区分升段/降段局。想启用段位检测的话，截取对应图片放入模板目录即可。

### 状态栏提示"截图失败"

**常见原因**：
- **BitBlt: 拒绝访问** — 游戏窗口权限变动（如从管理员权限启动）。关闭后重新从 Steam 启动
- **窗口关闭** — 对局结束后正常现象，程序会自动重试

### 检测准确率低 / 误识别

1. 模板是否在当前分辨率下截取？不同分辨率的 UI 像素位置不同
2. 模板是否截取了特征明显的 UI 区域（纯色块 > 小图标 > 高频复用的文字）
3. 在 `config.toml` 中适当调整 `detection.confidence_threshold`（降低 = 更敏感）

### 一直检测不到（硬币/先后攻/胜负始终不触发）

**典型场景**：游戏更新后字体或 UI 变了，旧模板失效；或者换显示器后分辨率不匹配。

**排查步骤**：

1. **开启调试功能**：设置 → 识别 → 调试设置 → 勾选「保存最佳失败样本」，配置文件中的 `debug.save_failure_samples` 设为 `true`
2. 置信度偏移量（`debug.failure_sample_offset`）控制记录范围——偏移量越大越容易触发记录，默认 0.10 通常够用
3. **正常玩游戏**：让程序跑几局。程序会把每一阶段中最接近成功的截图和诊断信息自动保存到 `screenshots/debug/` 目录
4. **分析失败样本**：打开 `screenshots/debug/`，每个 target 至多一个 PNG + TOML 文件：
   - 看 `[all_scores]` 中的各模板分数——**全部低分**说明整体环境有问题（分辨率不匹配/模板失效），**个别低分**说明该模板需更新
   - 看 `matched_template` —— 如果匹配到的模板名和预期不符，说明模板之间区分度不够
   - 看 `roi` 和 `window_rect` —— 排查搜索区域偏移和多显示器问题
5. **根据诊断结果修复**模板或调参，问题解决后关掉功能即可
6. `screenshots/debug/` 文件不会自动清理，排查完毕后可手动删除

### 段位图标检测突然失效（切换分辨率/窗口后）

**原因**：`resource/templates/rankicons/rank_positions.toml` 缓存的坐标基于旧窗口位置，窗口移动后可能偏移出搜索范围。

**解决**：删除 `resource/templates/rankicons/rank_positions.toml`，下次检测到段位图标时自动重新全图搜索并写入新缓存。同样，`resource/templates/{分辨率}/roi.toml` 也可删除重置。

## 界面显示

### 字体显示异常 / 缺字

在设置中切换到已安装的字体（如 Microsoft YaHei）即可。

### 主题切换后颜色不对

1. 确认 `config.toml` 中 `appearance.theme` 的文件夹名与 `themes/` 下一致
2. 点「设置 → 确定」后会自动重载主题

## 文件与数据

### CSV 文件打不开 / 乱码

- CSV 使用 UTF-8 编码，Excel 默认用 ANSI 打开会乱码
- 解决方案：Excel → 数据 → 从文本/CSV 导入 → 选择 UTF-8

### CSV 文件被其他程序占用（WPS/Excel 等）

**表现**：状态栏持续显示「⚠ 文件被占用，N 条暂存」，对局结束后记录没有写入 CSV。

**原因**：WPS、Excel、记事本等程序打开 CSV 时会锁定文件，程序无法写入。

**解决**：
- 关闭占用 CSV 的程序即可，下次对局结束自动补写所有暂存记录
- 退出程序时会弹出三选一窗口：`重试写入`（关掉占用程序后点） / `不退出`（留在程序里等自动补写） / `强制退出`（放弃暂存，需二次确认）
- 加载/备份/新建数据前也会弹出同样窗口
- 删除/修改记录时如文件被占用，状态栏提示「删除/修改失败 — CSV 文件被占用」

### 对局记录没有保存

1. 检查 `csv/` 目录是否有 `data.csv`（或按日期命名的文件）
2. 确认程序有写入权限（不要放在 C:\\Program Files 等受保护目录）
3. 如状态栏显示「⚠ 文件被占用」，参考上方「CSV 文件被其他程序占用」一节

### 段位图标检测不到

**表现**：对局结束后 CSV 的己方段位/对方段位列始终为空。

**检查步骤**：

1. 确认设置 → 识别 →「启用段位图标检测」已勾选
2. 确认 `resource/templates/rankicons/` 目录存在且包含 9 枚 PNG（8 枚段位图标 + crown）
3. 首次运行或切换分辨率后首次检测可能需 2~3 局才能命中（位置缓存尚未建立）
4. `resource/templates/rankicons/rank_positions.toml` 被误删：下次检测到段位图标时自动重新生成
5. 置信度阈值过高：在设置中降低段位置信度阈值（默认 0.7）可提高命中率，但可能引入误识别

**兜底**：段位检测失败不影响对局的硬币/先后攻/胜负记录，只缺少段位信息。

### config.toml 被误删了

程序首次启动时自动生成默认配置文件，不会崩溃。原有配置需重新设置。

### .app_state.toml 格式变化

每次升级如数据库结构有变化，旧格式自动兼容——缺的字段用默认值补齐，下次关闭时写回完整文件。旧版 `.app_state.json` 不会自动迁移，升级后窗口位置重置为默认值（仅一次）。

## 截图热键

### 热键按下没反应

1. 确认设置 → 识别 → "启用截图热键" 已勾选
2. 启动后状态栏应显示"截图热键已启用"
3. `Ctrl+Shift+S` 被其他软件（如浏览器、IDE）占用时注册失败，状态栏会提示。换一个组合键如 `Ctrl+Shift+F10`

### 热键注册失败

热键可能被其他程序占用。在设置中点击热键输入框，改为其他组合键（如 `Ctrl+Alt+字母`）。

## OBS 与直播

### OBS 窗口捕获找不到悬浮窗

悬浮窗默认使用 `Tool` 窗口类型（无任务栏图标），OBS 的窗口捕获会过滤掉此类窗口。

**解决**：设置 → 悬浮窗 → 勾选「OBS 捕获模式」，悬浮窗会显示任务栏图标，OBS 即可正常捕获。或者改用 OBS 的「显示器捕获」替代「窗口捕获」。

### OBS 抓到的是透明/空白画面

悬浮窗使用了半透明背景（`WA_TranslucentBackground`），部分 OBS 捕获模式对 Layered Window 兼容性不佳。

**解决**：设置 → 悬浮窗 → 关闭「使用主题背景图」并设置不透明背景色，或使用 OBS 的「显示器捕获」+ 颜色键过滤。

## 通知与托盘

### 对局结束没有弹出通知

1. 确认设置 → 系统 → "对局结束系统通知" 已勾选
2. Windows 通知设置中是否禁用了 MD Stats 的通知？（设置 → 系统 → 通知）
3. 全屏游戏时通知被遮挡 → 正常现象

### 最小化到托盘不生效 / 任务栏仍有图标

1. 确认设置 → 系统 → "最小化到系统托盘" 已勾选
2. 点击"最小化"按钮（窗口右上角）触发，不是直接按 Win+D

### 托盘图标不显示 / 右键菜单点击无反应

1. 右下角系统托盘可能折叠了图标 → 点击^展开
2. 托盘图标已显示但功能异常 → 重启程序

## 段位检测

### 段位检测不工作 / 状态栏无段位信息

段位图标源素材因版权原因未包含在仓库中，需自行准备：

1. 确认 `resource/templates/rankicons/` 下存在 9 张 RGBA PNG 图片
2. 图片需从 Master Duel 游戏资源包中用 AssetRipper 提取，详见 `resource/templates/README.md`
3. 缺失时段位检测自动静默跳过，不影响硬币/先后攻/胜负识别

### 段位识别结果不对（等级 I~V 错误）

1. I/II/III 通常准确率较高；IV 和 V 在低分辨率下可能互相误判
2. 可尝试调整 `config.toml` 中 `rank_detection.confidence_threshold`（默认 0.7，降低可提高识别率但可能误检）

## 性能

### 打包的 ZIP 有 130+ MB，为什么这么大

程序依赖 PySide6（Qt 框架，52 MB）+ OpenCV（101 MB）+ NumPy（27 MB）。这些都是必需的图像识别和 GUI 基础库，无法进一步瘦身。
