# 常见问题排查

> **最常见的识别问题根源是模板不匹配**——请确保模板图片在当前游戏分辨率下截取。
> 如硬币/先后攻/胜负识别不准，优先重截对应模板再试。

## 识别与检测

### 状态栏一直显示"等待 Master Duel 窗口…"，不开始识别

**原因**：程序无法找到或获取游戏的客户区尺寸。

**检查**：
1. Master Duel 是否已启动？窗口标题需要包含英文 `masterduel`（Yu-Gi-Oh! Master Duel）
2. 窗口是否最小化？恢复窗口再试
3. 分辨率是否 0×0？切换到窗口模式再切回全屏

### 启动时提示"缺少 N 个模板"，状态栏卡住

**原因**：`resource/templates/{分辨率}/` 下缺少必选模板（coin_win/coin_lose/go_first/go_second/victory/defeat）。

**解决**：在对应分辨率目录下放入全部 6 张必选模板。可选模板（rank_up/rank_down）缺失不影响启动。

### 提示"缺少段位检测模板: rank_up.png（已跳过，视为普通局）"

**正常行为**：rank_up/rank_down 是可选模板，缺失时自动跳过，对局照样记录，只是不区分升段/降段局。想启用段位检测的话，截取对应图片放入模板目录即可。

### 状态栏提示"截图失败"

**常见原因**：
- **BitBlt: 拒绝访问** — 游戏窗口权限变动（如从管理员权限启动）。关闭后重新从 Steam 启动
- **窗口关闭** — 对局结束后正常现象，程序会自动重试

### 检测准确率低 / 误识别

1. 模板是否在当前分辨率下截取？不同分辨率的 UI 像素位置不同
2. 模板是否截取了特征明显的 UI 区域（纯色块 > 小图标 > 高频复用的文字）
3. 在 `config.toml` 中适当调整 `detection.confidence_threshold`（降低 = 更敏感）

## 界面显示

### 字体显示异常 / 缺字

在设置中切换到已安装的字体（如 Microsoft YaHei）即可。

### 主题切换后颜色不对

1. 确认 `config.toml` 中 `appearance.theme` 的文件夹名与 `themes/` 下一致
2. 点「设置 → 确定」后会自动重载主题

## 文件与数据

### CSV 文件打不开 / 乱码

- CSV 使用 UTF-8 编码，Excel 默认用 ANSI 打开会乱码
- 解决方案：Excel → 数据 → 从文本/CSV 导入 → 选择 UTF-8

### 对局记录没有保存

1. 检查 `csv/` 目录是否有 `data.csv`（或按日期命名的文件）
2. 确认程序有写入权限（不要放在 C:\\Program Files 等受保护目录）
3. CSV 被 Excel 锁定时程序会静默失败 → 关闭 Excel 再试

### config.toml 被误删了

程序首次启动时自动生成默认配置文件，不会崩溃。原有配置需重新设置。

### .app_state.json 格式变化

每次升级如数据库结构有变化，旧格式自动兼容——缺的字段用默认值补齐，下次关闭时写回完整文件。

## OBS 与直播

### OBS 窗口捕获找不到悬浮窗

悬浮窗默认使用 `Tool` 窗口类型（无任务栏图标），OBS 的窗口捕获会过滤掉此类窗口。

**解决**：设置 → 悬浮窗 → 勾选「OBS 捕获模式」，悬浮窗会显示任务栏图标，OBS 即可正常捕获。或者改用 OBS 的「显示器捕获」替代「窗口捕获」。

### OBS 抓到的是透明/空白画面

悬浮窗使用了半透明背景（`WA_TranslucentBackground`），部分 OBS 捕获模式对 Layered Window 兼容性不佳。

**解决**：设置 → 悬浮窗 → 关闭「使用主题背景图」并设置不透明背景色，或使用 OBS 的「显示器捕获」+ 颜色键过滤。

## 通知与托盘

### 对局结束没有弹出通知

1. 确认设置 → 系统 → "对局结束系统通知" 已勾选
2. Windows 通知设置中是否禁用了 MD Stats 的通知？（设置 → 系统 → 通知）
3. 全屏游戏时通知被遮挡 → 正常现象

### 最小化到托盘不生效 / 任务栏仍有图标

1. 确认设置 → 系统 → "最小化到系统托盘" 已勾选
2. 点击"最小化"按钮（窗口右上角）触发，不是直接按 Win+D

### 托盘图标不显示 / 右键菜单点击无反应

1. 右下角系统托盘可能折叠了图标 → 点击^展开
2. 托盘图标已显示但功能异常 → 重启程序

## 性能

### 打包的 ZIP 有 130+ MB，为什么这么大

程序依赖 PySide6（Qt 框架，52 MB）+ OpenCV（101 MB）+ NumPy（27 MB）。这些都是必需的图像识别和 GUI 基础库，无法进一步瘦身。
