# Master Duel 段位识别研究

## 目标

在《游戏王 Master Duel》的 coin toss 阶段，识别双方头像旁显示的段位等级。

## 段位图标结构

段位由两部分组成：`rankicon_XX`（段位徽章） + `tier_X`（等级数字）。

已从 AssetRipper 解包获得的源素材：

| 类型 | 文件 | 尺寸 |
|------|------|------|
| rankicon_01~07 | 段位徽章 PNG | 128×128 / 290×290 |
| tier1~tier5 | 等级数字 PNG | ~12×32 / ~26×32 |
| crown | 王冠图标 | ~96×56 |
| tier | 通用 tier 标识 | ~26×32 |
| rateicon_01 | 评分赛图标 | ~290×290 |

截图中段位图标实际渲染尺寸约 **21~28 px**（1080p）。

## 实测结果

### 源素材直接匹配（全部失败）

| 方法 | 最高分 | 阈值 |
|------|:---:|:---:|
| 灰度 NCC | 0.39 | < 0.70 |
| 暗底合成 + 模糊 NCC | 0.56 | < 0.70 |
| Canny 边缘 NCC | 0.34 | < 0.70 |
| SSIM 结构相似 | 0.17 | < 0.50 |

### 原因分析

0.39~0.56 这个区间属于「不像完全错误，也不像正确」——说明模板内容对了但尺寸或采样方式不对。Unity Canvas Scaler 的缩放链路与 OpenCV `cv2.resize()` 不一致是最大疑点。

## 已验证失败的方向

- CNN/YOLO：项目太重，8 枚小图标不值得引入
- SIFT/ORB：21px 图片特征点太少
- 边缘匹配：源素材边缘与渲染画面不一致

## 已知可行但未采用的方向

- **截图裁剪做模板**：当前项目 coin_win/coin_lose 正是这种方案。顾虑是 8 枚图标 × 3 分辨率 = 45 张模板，且只有 1080p 显示器。

## 待验证

1. Unity Canvas Scaler 的参考分辨率和缩放模式（能否从资源文件中获取）
2. 不同 `cv2.resize` 插值方式（INTER_NEAREST/LINEAR/AREA/CUBIC/LANCZOS4）的效果差异
3. 是否存在某个尺寸 + 插值组合能显著提升匹配分
