# MD Stats 产品需求文档

## 概述

**产品名**: MD Stats  
**协议**: MIT  
**定位**: 基于图像识别的 Yu-Gi-Oh! Master Duel 对局自动统计工具  
**平台**: Windows  
**语言**: Python + PySide6 + OpenCV  
**仓库**: https://github.com/learbox/MD_Stats

## 核心功能

### 1. 自动识别对局信息

**用户故事**: 作为玩家，我希望程序在后台自动检测我的对局结果，这样我不用手动记录每一局的输赢。

**详细说明**:
- 定时截取 Master Duel 游戏窗口画面（用户可设间隔，默认 0.3 秒）
- 用 OpenCV 模板匹配识别三阶段信息：
  1. 硬币输赢 — 对局开始时猜硬币的结果，同时检测是否为升段/降段局（可选）
  2. 先后攻 — 硬币猜中/没猜中后的先后手
  3. 对局胜负 — 对局结束时的胜/负画面
- 三个阶段按顺序检测，前一阶段未识别到不检测后续阶段
- 对局结束自动写入 CSV 文件
- 支持用户自行截取模板图片，按游戏分辨率分目录存放

**验收标准**:
- 给定正确的模板图片，识别准确率 > 95%
- 识别到结果后在状态栏显示消息
- 自动写入 CSV（含己方/对方段位等级），无需用户额外操作
### 2. 段位图标检测

**用户故事**: 作为玩家，我希望程序在对局中自动识别双方段位等级，无需手动输入。

**详细说明**:
- 独立线程运行，与主管线解耦，不影响三阶段检测
- 通过 AssetRipper 解包获得 RGBA 源素材，无需截图制作模板
- 背景色自适应合成，匹配游戏 UI 实际渲染画面
- 检测双方段位写入 CSV 己方段位/对方段位列
- 支持 8 种段位：新手、青铜、白银、黄金、铂金、钻石、大师、巅峰
- 等级数字 I~V 通过连通组件识别（巅峰无数字等级）
- 可配置截图间隔（默认 0.5 秒）和置信度阈值（默认 0.7）
- 可在设置中关闭

**验收标准**:
- 给定正确的源素材，段位识别准确率 > 90%
- 检测到后状态栏显示"段位: 铂金 vs 钻石"
- 检测不到时不报错、不影响主流程

### 3. 手动录入

**用户故事**: 作为玩家，我希望在自动识别遗漏时能手动补录，并且手动和自动不冲突。

**详细说明**:
- 两个手动按钮的文字和颜色随对局阶段变化（硬币=橙色/先后攻=蓝色/胜负=绿红）
- 左按钮始终代表"赢/先攻/胜"，右按钮代表"输/后攻/负"
- 手动录入和自动识别共用一个三阶段状态机，手动操作后自动识别从当前阶段继续
- 支持逐级撤销（从阶段 2 回退到阶段 1，再回退到阶段 0）

### 4. 数据展示

**用户故事**: 作为玩家，我希望直观看到各卡组对局的统计数据。

**统计表格**:
- 按卡组汇总：对局数、胜、负、胜率、赢/输硬币次数、硬币胜率、先后攻数据
- 最后一行显示合计
- 所有百分比数值

**记录表格**:
- 每条对局的详细信息，最新在前
- 支持单元格编辑（下拉菜单 + 自由输入），修改自动写回 CSV
- 支持删除最后一条记录（需确认）

### 5. 悬浮统计窗

**用户故事**: 作为主播/多屏玩家，我希望有一个单独的小窗显示当前卡组数据。

**详细说明**:
- 无边框、半透明、始终置顶、可拖拽
- 支持合并显示（如"胜/负"合并为一行）
- 通过 `floating_window.rows` 配置自定义显示哪些数据行
- 切换 `floating_window.use_theme_bg` 可在纯色（便于 OBS 色键捕捉）和主题背景图之间切换
- 背景色、不透明度、字体、字号、文字颜色均可配置
- 位置自动持久化到 .app_state.json
- 主窗口最小化时悬浮窗不受影响

### 6. 主题系统

**用户故事**: 作为用户，我希望能自定义界面外观，或者选预置的暗色/亮色主题。

**详细说明**:
- 预置三套主题: 暗色沉浸、亮色清爽、马卡龙水彩
- 用户可自定义主题: 在 themes/ 下新建文件夹，修改 theme.toml 颜色和图片
- 支持纯色和背景图两种模式
- 支持自定义 TTF/OTF 字体
- 切换主题即时生效（点击重载配置）
- themes 文件夹缺失时自动回退内置亮色主题

### 7. 图形化设置

**用户故事**: 作为不熟悉写代码的用户，我希望能用图形界面修改配置，不用打开文本文件。

**详细说明**:
- 主界面"设置"按钮打开弹窗
- 6 个标签页覆盖全部配置项（识别/外观/剪贴板/悬浮窗/数据/系统）
- 颜色用取色器选择（不需要手写 #RRGGBB）
- 字体、字号、透明度等都有直观控件
- 所有输入控件自带范围校验
- 点确定后自动写回配置文件并重载

### 8. 剪贴板导出

**用户故事**: 作为玩家，我希望能把统计数据复制到剪贴板，贴到聊天或文档里。

**详细说明**:
- 横排 TSV 格式（可粘贴到 Excel / 表格）
- 竖排格式：`[卡组名]\nkey\tvalue\n...`（方便阅读）
- `clipboard.scope` 选择范围：`"current"` = 当前卡组，`"all"` = 全部卡组
- `clipboard.columns` 选择复制哪些列（空 = 全部）
- 以上所有配置可通过设置弹窗可视化编辑

### 9. 版本更新检查

**用户故事**: 作为用户，我希望不用打开浏览器就能知道有没有新版本。

**详细说明**:
- "关于"弹窗中点击「检查更新」按钮
- 查询 GitHub Releases API 获取最新版本号
- 比较三段式版本号（不会把 1.10 误判为小于 1.9）
- 分别显示：有新版本 / 已是最新 / 网络失败

### 10. 配置持久化

**用户故事**: 作为用户，我希望下次打开软件时，窗口位置和列宽和上次一样。

**详细说明**:
- 记住主窗口位置和大小
- 记住统计表格和记录表格的列宽
- 记住表格分割线位置（用户自行拖拽调整）
- 记住悬浮窗位置
- 全部存在 `.app_state.json` 中
- 默认值集中管理（`_APP_STATE_DEFAULTS`），升级时缺字段自动补齐
- 记录表格的列宽跳过隐藏列（序号）和自动填充列（备注），仅保存有效列宽

### 11. 调试工具

**用户故事**: 作为用户，我希望在出现问题时能查看日志和检测截图，方便排查。我也希望用热键快速截取游戏画面来制作模板。

**详细说明**:
- **保存检测截图**: 开启后在每次检测到硬币/先后攻/胜负时自动保存截图到 `screenshots/` 目录，文件名包含检测类型、分辨率和时间戳。下一局开始时自动清除上一局的截图
- **截图热键**: 两个全局热键（可自定义），游戏全屏时也能响应。单次截图（Ctrl+Shift+S）按一次截一张；周期截图（Ctrl+Shift+D）按一下开始定时截、再按停止
- **日志模式**: 开启后将程序运行信息写入 `logs/` 日志文件
- **日志作用域选择**: 用户可选择记录哪些内容——状态栏消息、截图事件、错误信息
- **全局异常捕获**: 未处理的异常自动写入日志（如配置文件读取失败）
- **工作线程异常捕获**: 后台识别线程的异常也写入日志（QThread 不受 sys.excepthook 覆盖）
- 设置弹窗中一键「查看日志」打开日志文件夹

**验收标准**:
- 开启截图保存后，每局对局生成 3 张检测截图
- 开启日志模式后，状态栏消息写入日志文件并带时间戳
- 日志作用域切换即时生效
- 异常信息能出现在日志文件中
- 热键注册成功后状态栏提示，按下即截取游戏画面

**详细说明**:

### 12. 系统通知与托盘

**用户故事**: 作为玩家，我希望对局结束时能收到通知，不用切出游戏查看结果。

**详细说明**:
- 对局结束时弹出系统气泡通知，显示硬币输赢（含段位升降）、先后手和胜负结果
- 通知显示时长可配置（1-30 秒）
- 支持最小化到系统托盘（不占任务栏），悬浮窗不受影响
- 托盘图标右键菜单：显示窗口 / 退出程序

**验收标准**:
- 开启后对局结束弹出 Windows 系统通知
- 托盘图标始终可见
- 最小化到托盘不影响检测和悬浮窗

## 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 启动时间 < 1 秒，占用内存 < 200MB |
| 可靠性 | 截图失败不崩溃，窗口未检测到时自动重试 |
| 兼容性 | Windows 10/11 |
| 可用性 | 无需命令行知识，双击 EXE 即可运行 |

## 不包含的功能

- 移动端（手机）
- macOS / Linux（Master Duel 目前仅支持 Windows）
- 对战录像
- 在线排行榜
- 多语言（仅简体中文）
- Steam Deck / Wine 兼容
